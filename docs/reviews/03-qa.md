# Review 3: QA Tester

## CRITICAL Issues

1. **Product ID mismatch breaks selection** - `src/app/(onboarding)/select-products/page.tsx:15`
   - defaultSelected=['yeast_mandazi', 'doughnuts', 'loaf_1kg', 'loaf_500g', 'chapati']
   - PRODUCTS array has: 'yeast-mandazi', 'daddies', 'italian-cookies'
   - Confirm-prices page won't find these products

2. **0 kg entry allowed but behavior unclear** - `src/app/(main)/entry/page.tsx:52-56`
   - Input allows min="0"
   - When kgFlour=0, auto-calculated values section disappears silently (line 78)
   - No warning that 0 treated as "no entry"

3. **Empty production/sales can be saved** - `src/app/(main)/entry/page.tsx:250-280`
   - No validation prevents saving all zeros
   - Save button never disabled for empty entries

4. **No bakery validation before save** - `src/app/(main)/entry/page.tsx:289-295`
   - Checks `if (!onboardingData.bakery)` only on save
   - User can input data then see error

## HIGH Issues

1. **Negative numbers can bypass min="0"** - `src/app/(main)/entry/page.tsx:52,135`
   - Mobile users can type negative manually
   - Users can paste negative values

2. **Browser back loses data** - No route guards
   - User on /entry can tap back and lose all data
   - No unsaved changes detection

3. **Deep linking without bakery** - `src/app/(main)/entry/page.tsx:289`
   - Can navigate to `/entry` before onboarding
   - Shows "No bakery" but allows entry attempts

4. **localStorage cleared = all data lost**
   - If user clears browser cache, all local entries gone
   - No warning or backup

## MEDIUM Issues

1. **Decimal allows very small numbers** - `src/app/(main)/entry/page.tsx:51`
   - step="0.5" allows 0.5, 1.5
   - 0.0001 kg may be accepted

2. **Large numbers may overflow UI** - `src/app/(main)/entry/page.tsx:139`
   - No max limit on sales input
   - 999999999 may cause display issues

3. **Tab switching doesn't auto-save** - entry/page.tsx
   - Intermediate values only in state until Save clicked

4. **Form not disabled during save** - `src/app/(main)/entry/page.tsx:240`
   - User can modify inputs while saving
   - Race condition possible

## LOW Issues

1. **localStorage key collisions possible** - entry/page.tsx:302
   - Mitigated by using controlled bakery IDs

2. **No Firestore/localStorage conflict resolution**
   - Edits from different devices may conflict

3. **Toast notification may be missed**
   - If user on another tab, toast appears off-screen
